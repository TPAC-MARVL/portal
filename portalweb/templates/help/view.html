Grid Definition

On the right side of the page:
  • : click to check the spatial coverage of the available forcing datasets. Available forcing dataset are:
    • Access-a surface
    • NCEP (National Center for Environmental Prediction)
    • BRAN3 (Bluelink Reanalysis)
    • FOAM1 (Forecasting Ocean Assimilation Model)
    • OFAM2 (Ocean Forecasting Australia Model)
  • :set the grid shape. The default shape is rectangular. Option for a curvilinear grid are available for ROMS. Only the horizontal resolution can be parameterized. The vertical grid has a default number of xx levels
  • The parameterisation of the horizontal grid can be achieved in two ways:
    • horizontal extent and rotation can be defined by clicking and dragging on the map (A)
    • parameter value (origin, size, spacing, rotation) can be entered in the box situated on the right hand panel (B):
      • Origin: location on the map of the large white circle. From this point the grid is defined using the parameters size, spacing and rotation
      • Size: set the number of grid rows and columns
      • Spacing: set longitudinal and latitudinal grid lengths in degrees. If a explicit numerical scheme is used, care should be taken to adjust the model timestep/horizontal resolution to ensure the model numerical stability by fulfilling the CFL criterion
      • Rotation: rotation is anti-clockwise i.e. +10 degrees is 10˚ anti-clockwise
  • Pressto update the grid parameterisation
